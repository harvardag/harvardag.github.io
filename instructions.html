Instructions for AlumniMagnet Technical Team

Executive Summary

The purpose of this repository is to give the existing AlumniMagnet codebase a clean modern look and also to make the site almost entirely mobile-responsive. This is accomplished by adjusting the header and footer of the standard page template and by adding a new set of CSS stylesheets. That's it -- none of the page content html is changed.

The following pages are mobile-responsive. All content adjusts to fit the viewport width on smaller screens:

  • index.html
  • article.html
  • news.html
  • events.html
  • featured_members.html
  • sponsors.html
  • volunteers.html
  • partners.html

Some pages, however, such as the directory, forums, and admin pages, have content that is not width-adjustable. For these pages we simply add an extra stylesheet in the footer that loads "static.css", and, voila, the page contents never get resized smaller than the appropriate width. This width is 992px on the admin pages and 808px on the other pages. In summary, the following pages are non-responsive:

  • admin.html
  • people.html
  • forum.html
  • myaccount.html
  • any others as appropriate

Finally, to give each page a clean, modern look,, certain pages get their own stylesheet inserted in the footer of the page. These are:

  • admin.html, article.html, events.html, forum.html, myaccount.html, news.html, and people.html.

Technical Overview

The new site theme differs from the existing one as follows. On the current standard-issue portal page, all page content is wrapped inside columns of a table. It looks like this.


<!-- Header -->
	<table id="main_stage" ...>
		<tr>
			<td ...>...</td>
			<td ...>
				LEFT CONTENT HERE
			</td>
			<td ...>
				CENTER CONTENT HERE
			</td>
			<td ...>
				RIGHT CONTENT HERE
			</td>			
		</tr>
	</table>
<!-- Footer -->

In the new design, we use divs instead. Note that the center content goes first, then the left, then the right. This is so search engines and older browsers see the content in the right order. Notice that the only difference is that the <dt> tags are replaced by <div> tags:


<!-- Header -->
	<div class="portal-wrapper">
		<div id="portal-center-wrapper" class="portal-center-wrapper">
			CENTER CONTENT HERE
		</div>
		<div id="portal-left-wrapper" class="portal-left-wrapper">
			LEFT CONTENT HERE
		</div>
		<div id="portal-right-wrapper" class="portal-right-wrapper">
			RIGHT CONTENT HERE
		</div>
	</div>
<!-- Footer -->

On pages with two columns instead of three (i.e. sponsors.html, featured_members.html, volunteers.html, events.html, news.html) we do a similar thing:


<!-- Header -->
	<div class="double-wrapper">
		<div id="double-center-wrapper" class="double-center-wrapper">
			CENTER CONTENT HERE
		</div>
		<div id="double-right-wrapper" class="double-right-wrapper">
			RIGHT CONTENT HERE
		</div>
	</div>
<!-- Footer -->

Finally on "regular" pages like article.html, we have a single column of content. For these pages, content gets put directly in between the header and footer:


<!-- Header -->
			ARTICLE CONTENT HERE
<!-- Footer -->

Page content

All content in the new design has exactly the same html as on the existing site, with the following exceptions:

Events:

  • We love how events look at http://www.hbsab.org. In particular, the html for displaying events in the MIDDLE of the page, with the big image of the date of the event, we'd like to use that on our site. But the code for the way events show up on sidebars, and the actual html for the calendar on the upcoming events page, we'd like to keep everything how it currently is on our site.

Sign In Page:

  • Similarly, we love how the sign in page looks at http://www.hbsab.org/user.html?op=login&nonalum and http://www.hbsab.org/user.html?op=login. We'd like to get something that looks like this because it is way more intuitive than what we currently have.

News Page:

  • Finally, we'd like to adopt the "Archive" column on the news page like it is at http://www.hbsab.org/news.html.

All other content:

  • Stays exactly the same as it is.

Mobile site

Currently when viewing from a mobile device, the browser automatically goes to a separate, limited version of the site at mobile.html. With the new design, there is only one version of the site and the extra mobile page is no longer necessary.

Next Steps

To load the new design, the following steps are needed:

  • Update the header and footer according to header.txt and footer.txt.
  • For the portal page and pages that have two columns of content, wrap the content in <div> tags instead of <td> tags. That's it.
  • Upload the folders "assets" and "dist" to the root directory of tht site. These contain all the new CSS and appropriate JS.
  • Make sure that the following pages get links to their own stylesheets in the footer:
    • admin.html
    • events.html
    • forum.html
    • people.html
    • myaccount.html.
  • The main navigation menu follows the standard Bootstrap 3 format instead of tables.
  • That's it!!

Links to files

All files and links in this design point either to a full http address, or to a relative address relative to the root directory of the website. If the folders "assets" and "dist" are uploaded to the root directory, no recoding of links is necessary.

CSS

All CSS is either in the bootstrap core, or in the files in assets/css, or in the custom CSS the AlumniMagnet automatically loads in certain pages such as admin.html or myaccount.html. The only thing that needs to be updated is the background image for the site. Simply make sure "assets" and "dist" are in the root directory and all CSS gets automatically loaded.
